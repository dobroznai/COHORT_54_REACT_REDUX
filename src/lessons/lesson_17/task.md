# Задание: Приложение Employees

## Общие требования

Создать приложение Employees со следующими страницами:

- "Create Employee" - для создания новых сотрудников
- "Employees" - для просмотра списка всех сотрудников

## Структура приложения

### Layout

- Должен включать **Header** и **Main** части
- В Header должны быть 2 навигационные ссылки: "Create Employee" и "Employees"
- При клике на ссылку, в Main части отображается соответствующая страница

## Страница "Create Employee"

### Форма создания сотрудника

#### Поля формы:

**Name\*** (обязательное)

- Компонент: Input
- Тип данных: string
- Мин. символов: 2
- Макс. символов: 50

**Surname\*** (обязательное)

- Компонент: Input
- Тип данных: string
- Макс. символов: 15

**Age\*** (обязательное)

- Компонент: Input
- Тип данных: string
- Мин. символов: 1
- Макс. символов: 3

**Job Position** (опциональное)

- Компонент: Input
- Макс. символов: 30

#### Кнопка "Create" (submit)

При клике на кнопку:

1. Происходит валидация полей формы
2. Если есть ошибки - они показываются под соответствующими полями, карточка не создается
3. Если ошибок нет - данные сохраняются в Redux state и на странице "Employees" создается карточка с этими данными

## Страница "Employees"

- Отображает все карточки сотрудников, которые были созданы через форму "Create Employee"
- Все данные карточек сохраняются в Redux state
- Логика удаления, получения и изменения данных контролируется через Redux

## Технологический стек

- **Formik** - для управления формой
- **Yup** - для валидации данных
- **Redux** - для управления глобальным состоянием
